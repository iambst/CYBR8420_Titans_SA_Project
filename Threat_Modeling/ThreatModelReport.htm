Interaction: Customer Data Request

1. An adversary may leverage the lack of monitoring systems and trigger anamolous traffic to database  [State: Mitigation Implemented]  [Priority: High] 

Category:Tampering
Description:An adversary may leverage the lack of intrusion detection and prevention of anomalous database activities and trigger anamolous traffic to database
Justification:NextCloud logs all database access. Using this logs anamolous traffic is detected and tampering is prevented.
SDL Phase:Design

2. An adversary can tamper critical database securables and deny the action  [State: Needs Investigation]  [Priority: High] 

Category:Tampering
Description:An adversary can tamper critical database securables and deny the action
Justification:No digital signature is provided to the critical database securables and there is a risk of rogue user to tamper the critical database securables.
SDL Phase:Design

3. An adversary can deny actions on database due to lack of auditing  [State: Mitigation Implemented]  [Priority: Medium] 

Category:Repudiation
Description:Proper logging of all security events and user actions builds traceability in a system and denies any possible repudiation issues. In the absence of proper auditing and logging controls, it would become impossible to implement any accountability in a system.
Justification:NextCloud logs data in the nextcloud.log file provided in the root directory. Full audit trial can be recorded. This prevents repudiation.
SDL Phase:Implementation

4. An adversary can gain access to sensitive data by performing SQL injection  [State: Needs Investigation]  [Priority: High] 

Category:Information Disclosure
Description:SQL injection is an attack in which malicious code is inserted into strings that are later passed to an instance of SQL Server for parsing and execution. The primary form of SQL injection consists of direct insertion of code into user-input variables that are concatenated with SQL commands and executed. A less direct attack injects malicious code into strings that are destined for storage in a table or as metadata. When the stored strings are subsequently concatenated into a dynamic SQL command, the malicious code is executed.
Justification:NextCloud uses prepared queries for retrieving data. But it doesnot provide any mechanism to completely stop SQL injection.
SDL Phase:Implementation

5. An adversary can gain access to sensitive PII or HBI data in database  [State: Needs Investigation]  [Priority: High] 

Category:Information Disclosure
Description:Additional controls like Transparent Data Encryption, Column Level Encryption, EKM etc. provide additional protection mechanism to high value PII or HBI data.
Justification:NextCloud provides end-to-end data encryption. However Data Encryption at column level in database needs to be implemented.
SDL Phase:Implementation

6. An adversary can gain unauthorized access to database due to loose authorization rules  [State: Not Applicable]  [Priority: High] 

Category:Elevation of Privileges
Description:Database access should be configured with roles and privilege based on least privilege and need to know principle.
Justification:NextCloud administrators are considered to be trusted. Hence this threat is not applicable.
SDL Phase:Implementation

7. An adversary can gain unauthorized access to database due to lack of network access protection  [State: Mitigation Implemented]  [Priority: High] 

Category:Elevation of Privileges
Description:If there is no restriction at network or host firewall level, to access the database then anyone can attempt to connect to the database from an unauthorized location
Justification:This threat is mitigated by providing File Access Control app which acts as a firewall and prevents elevation of privileges.
SDL Phase:Implementation
